Introduction

          7 + 7 #This is R code
          ```

Libraries

  • dplyr
  • readr
  • tidyverse

Data Types

  • Numeric: Any number with or without a decimal point: 23, 0.23, and the numeric null value NA
  • Character: Any grouping of characters on your keyboard (letters, numbers, spaces, symbols, etc) or text. Most strings are surrounded by single quotes ‘ … ‘. Also can be referred to as a “string.”
  • Logical: Only has two possible values - either TRUE or FALSE (without quotes).
  • Vectors: a list of related data types that is all the same type
  • NA: represents the absence of a value and is represented by the keyword NA (without quotes)

Variables

  • Variables allows us to store information and associate that information with a name
  • Assign variables with an arrow sign (a carat and a dash) <-

    Ex: full_name <- “First Last”

Vectors

  • List like structure
    • Ex: spring_months <- c(“March”, “April”, “May”, “June”)
  • Created by separating four character strings with a comma and wrapped inside c() (which is a combine function)
  • A few things you should know how to do with vectors:
    • You can check the type of element in a vector by using typeof(vector_name)
    • Check the length of a vector by using length(vector_name)
    • Access individual elements in the vector by using [] and placing the element position inside the brackets. Start counting elements at one NOT zero

Conditionals

  • Run a task based on certain conditions
  • If/Else statements are helpful:
    • Ex: if (TRUE) {
      print(“Go to sleep!”)
      } else {
      print(“Wake up!”)
      }

Logical Operators

  • Three logical operators:
    • The and operator (&)
    • The or operator (|)
    • The not operator (!)
    • The not operator reverses the value of a TRUE value and vice versa

Functions

  • We can call or use a function by stating the name of the function and following it with an opening and closing parentheses ie functionName()
  • Between the parenthesis, we usually pass an argument or a value that the function uses to conduct an action ie functionName(value)

Importing Packages

  • Need to run the command the first time you install a package: install.packages(‘package-name’)
  • To import a package: library(package-name)

Data Frames

CSVs

  • one of the most common formats for big datasets
  • the first row of a CSV contains column headings. All subsequent rows contain values.
  • each column heading and each variable is separated by a comma
  • loading and saving
    • df <- read_csv('file.csv')
    • df = the data frame, creates a data name

Data Frame

  • A data frame is a R object that stores tabular data in a table structur made up of rows and columns
  • if the data frame is small, you can display it by typing its name
  • the head() function returns the first 6 rows of a data frame
    • You can pass additional argument n to head() ie head(df, 8) to show first 8 rows
  • The function summary() will return summary statistics such as mean, median, minimum, maximum for each numeric column while providing class and length non-numeric columns

Piping

  • the pipe operator %>% helps increase the readability of data frame code by piping the value on its left into the first argument of the function that follows it. For example:
    • df %>%
    • head()
    • Pipes the data frame into the first argument of head() becoming:
    • head(df)

Selecting and Excluding Columns

  • you can select appropriate columns for analysis with the select()
    • Ex: select(customers, age, gender)
    • select() function makes a data frames as its first argument
    • all additional arguments are the desired columns to select
  • you can simplify the readability of your code using the pipe:
      • customers %>% select(age, gender)
    • this can be read as: from the customers table,

Filtering Rows with Logic

  • Subset a data frame by rows using dplyr's filter()
    • Ex: filter data by orders with customer's first name Joyce
      • orders %>% filter(first_name == 'Joyce')
  • For multiple arguments, use a comma in between each condition
  • To use the OR | operator:
    • orders %>% filter(shoe_type == 'clogs' | price < 20)
  • To use the not or bang ! operator:
    • Ex: find all the orders with shoe color NOT red
    • orders %>% filter(!(shoe_color == red))
  • Sort rows of a data frame in ascending order by the column provided as an argument
    • use the arrange() using dplyr
    • numeric columns: sorts lower to higher numbers
    • character: sorts by alphabetical order from A to Z
    • customers %>% arrange(name)
  • sort rows by descending order
    • customers %>% arrange(desc(age))
  • If multiple arguments are provided to arrange() it will order the rows by the column given as the first argument and use the additional columns to break ties in the values of the preceding columns
